# QA Report Tool — Windows 开机自启动

适合把本机当「常驻日报服务」时使用。目标：开机（或登录）后自动跑：

```text
uvicorn backend.app.main:app --host 0.0.0.0 --port 8000 --workers 1
```

推荐用 **任务计划程序**（下面方案 A）。不要用「启动」文件夹挂一个会弹黑窗的 bat（容易被关掉、也难看日志）。

---

## 方案 A：任务计划（推荐）

### 1. 确认能手动启动

在项目根目录执行一次，确认页面可开：

```powershell
cd C:\Users\13746\projects\qa-report-tool
.\scripts\start_server.ps1
```

浏览器访问：`http://127.0.0.1:8000/`

### 2. 一键注册开机任务

**以管理员打开 PowerShell**，执行：

```powershell
cd C:\Users\13746\projects\qa-report-tool
.\scripts\register_autostart.ps1
```

默认：当前用户**登录时**启动（`AtLogOn`），一般**不需要**管理员；任务名 `QA-Report-Tool`。

若出现 `拒绝访问`：不要用 `-AtStartup`，先跑上面的默认命令；或右键 PowerShell → **以管理员身份运行** 后再注册。

可选参数：

```powershell
# 开机即启（必须管理员 PowerShell；开机后等约 1 分钟再启服务）
.\scripts\register_autostart.ps1 -AtStartup
```

### 3. 验证

```powershell
# 看任务是否存在
Get-ScheduledTask -TaskName "QA-Report-Tool"

# 立刻试跑一次（不必重启）
Start-ScheduledTask -TaskName "QA-Report-Tool"

# 看日志
Get-Content C:\Users\13746\projects\qa-report-tool\backend\data\uvicorn.log -Tail 30
```

注销再登录，或重启电脑后，访问 `http://本机IP:8000/` 应仍可打开。

### 4. 取消自启

```powershell
Unregister-ScheduledTask -TaskName "QA-Report-Tool" -Confirm:$false
```

或：`Win + R` → `taskschd.msc` → 找到 `QA-Report-Tool` → 删除。

## 方案 B：图形界面自己建任务

1. `Win + R` → `taskschd.msc`
2. **创建基本任务** → 名称：`QA-Report-Tool`
3. 触发器：**当用户登录时**（或「计算机启动时」）
4. 操作：**启动程序**
   - 程序：`powershell.exe`
   - 参数：
     ```text
     -NoProfile -ExecutionPolicy Bypass -File "C:\Users\13746\projects\qa-report-tool\scripts\start_server.ps1"
     ```
5. 勾选「使用最高权限运行」（若公司策略需要）
6. 属性里建议：
   - 「如果任务失败，重新启动」可开
   - 「只有在计算机使用交流电源时才启动」可按笔记本情况关掉

## 注意

| 项 | 说明 |
|----|------|
| 机器要常开 | 关机 / 休眠后定时发信不会跑 |
| `--workers 1` | 脚本已固定；多 worker 会导致定时任务重复发 |
| 端口占用 | `start_server.ps1` 若发现 8000 已在听，会跳过再启，避免起两份 |
| 日志 | `backend/data/uvicorn.log` |
| 改代码后 | 需结束旧进程再启动，或重启任务：`Stop-ScheduledTask` 不适用杀进程时，可在任务管理器结束 `python`/`uvicorn`，再 `Start-ScheduledTask` |
| 路径 | 若项目不在 `C:\Users\13746\projects\qa-report-tool`，改脚本内路径或在脚本同目录调用即可（脚本按自身位置定位仓库根） |

## 手动停服

```powershell
# 结束占用 8000 的进程
$conn = Get-NetTCPConnection -LocalPort 8000 -State Listen -ErrorAction SilentlyContinue | Select-Object -First 1
if ($conn) { Stop-Process -Id $conn.OwningProcess -Force }
```
