全局注意事项（全员必读）

1. 迭代一开始就要在 MeterSphere 建子计划

每个 Sprint 开始后，尽快在 MeterSphere 对应 Module（与 Sprint 同名，如 OBIS-20260706-20260717）下为相关 Story 创建子计划（TEST_PLAN），并持续执行用例。

日报主表的每一行来自 MS 子计划；没有子计划 = 日报里不会出现这条 Story。

2. 子计划名称尽量与飞书 Story 一致（支持模糊匹配）

要求说明
推荐做法与飞书 User Story 标题尽量一致（从飞书复制最稳）
工具容错不区分大小写；全半角/多余空格会规范化；去掉 MS 前缀如 [100274]；相似度 ≥ 88% 也可匹配
仍可能对不上改名过多、关键词差太多，或两条 Story 太像（歧义时为防错挂会放弃匹配）
为何仍要对齐匹配失败 → 该行 Status / Ready 为空

组员每位测试同学

负责自己 Story / Bug 的提测与质量侧数据，以及把阻塞及时同步给发信人。

要维护什么

数据在哪里做什么
MeterSphere 子计划 + 执行MeterSphere迭代开始后创建子计划（名尽量=飞书 Story）；执行用例，保证 Passed/Failed/Blocked/No Run 真实
Ready for Test（提测）「编辑 Ready for Test」弹窗 → 保存 Ready提测后核对/修正 Yes/No、Ready Date、Comment；与 Reopen 分入口，互不覆盖
Bug Reopen「编辑 Bug Reopen」弹窗 → 保存 Reopen维护 Priority / Status / Summary / Times；与 Ready 分入口，互不覆盖
阻塞点 / Risk同步给发信人把阻塞、风险说清楚；由发信人汇总进页面的 Risk/Block
Ready / Reopen 保存后，同 Sprint 的预览、发信、定时都会用。清空某 Story 的 Ready 三列并保存 → 该行恢复用飞书推导。

组员自查

  • 我的 Story 在 MS 有子计划（名称接近飞书、Module 正确）
  • 执行状态（尤其 Blocked / No Run）正确
  • 已提测的 Story，Ready 信息已保存或已确认无误
  • Reopen Bug 已维护（如有）
  • 阻塞/风险已同步给发信人
  • 改完看过预览

发信人轮值 / 当日发日报的人

负责把日报「凑齐、核对、发出」。重点维护下面三类信息。

要维护什么

数据在哪里做什么
Test ENV主页「Sprint 维护数据」填写当前测试环境（如 SIT，UAT1，UAT2）→ 保存 ENV/Risk
Risk / Block同上审核并整理全员反馈的阻塞/风险，写成对外可读的说明 → 保存
收件人（To / CC）「收件人」按钮维护默认收件人并保存；发信时可临时改，临时改不会写回默认

建议操作顺序

  1. Module 选对当前 Sprint → 刷新预览
  2. 填 / 改 Test ENV，审核 Risk/Block → 保存 ENV/Risk
  3. 必要时分别打开「编辑 Ready for Test」/「编辑 Bug Reopen」做最后核对并各自保存
  4. 再 刷新预览，确认主表、Bug、ENV、Risk 无误
  5. 手动发送，或确认定时任务已启用且收件人正确

发信人检查清单

  • MS 侧子计划已建齐、名称与飞书足够接近（抽查预览 Status/Ready）
  • Test ENV 已保存
  • Risk/Block 已审核并保存
  • 收件人正确
  • 预览已刷新且数字、Ready、Bug 看起来合理
  • 再点发送

页面入口速查（两角色共用）

按钮 / 区域谁常用用途
Module + 刷新预览全员选 Sprint、核对日报
保存 ENV/Risk发信人持久化环境与风险说明
编辑 Ready for Test组员为主，发信人可终审仅维护 Ready（独立弹窗）
编辑 Bug Reopen组员为主，发信人可终审仅维护 Reopen（独立弹窗）
收件人发信人默认 To/CC（仅此处保存默认）
手动发送发信人立即发信（会重拉 MS）
定时任务发信人按星期/时刻自动发

数据从哪来（对照）

MeterSphere 子计划执行 ──┐
飞书快照（Story/Bug）  ──┼──→ 合并 → 预览 / 邮件
页面人工维护           ──┘
         │
         ├─ 组员：Ready for Test、Bug Reopen；（阻塞点同步给发信人写进 Risk）
         └─ 发信人：Test ENV、Risk/Block、收件人

常见问题

预览没有我的 Story？ → MS 未建子计划，或名称与飞书差太多，或 Module 不对。

Ready 和飞书不一致？ → 可能已人工保存（人工优先）；清空三列保存可恢复飞书。

改完 Risk/Ready 邮件还是旧的？ → 是否点了保存？是否刷新了预览？

默认收件人为何没变？ → 只在「收件人设置」里保存才会改默认；发送弹窗里的修改是一次性的。
